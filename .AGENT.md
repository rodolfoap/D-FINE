## Project overview

D-FINE is an official PyTorch implementation of the paper "Redefine Regression Task of DETRs as Fine-grained Distribution Refinement" (arXiv:2410.13842). It's a real-time object detection framework built on DETRs (Detection Transformers) that redefines bounding box regression as Fine-grained Distribution Refinement (FDR) and includes Global Optimal Localization Self-Distillation (GO-LSD). Models range from D-FINE-N (4M params, 42.8% AP) to D-FINE-X (62M params, 55.8% AP on COCO).

## Architecture

```
Input Image
    ↓
Backbone (HGNetv2: N/S/M/L/X)
    ↓
Hybrid Encoder (encoder mixes backbone features)
    ↓
D-FINE Decoder (transformer-based, FDR + GO-LSD)
    ↓
Postprocessor (box decoding)
    ↓
Detection Output (boxes + classes + scores)
```

**Key components:**
- **Backbone**: HGNetv2 family (B0-B5) — produces multi-scale feature maps
- **Encoder**: Hybrid encoder fuses backbone features via cross-scale attention
- **Decoder**: Transformer decoder with fine-grained distribution refinement for regression
- **Criterion**: Custom loss combining classification and distribution-based box regression
- **Matcher**: Hungarian matcher for bipartite matching of predictions to ground truth

## Conventions

- **Config merging**: Use `__include__` directive in YAML to merge base configs (dataset, runtime, optimizer)
- **Registry pattern**: `@register()` decorator for auto-registration of models, losses, data transforms
- **DDP training**: Uses `torchrun` for multi-GPU distributed training; rank-0 prints config
- **Class names**: PascalCase for classes; lowercase for functions
- **Path patterns**: Regex in optimizer config targets specific module groups (`^(?=.*backbone).*` for backbone-only)
- **Model zoo**: Configs follow naming `dfine_hgnetv2_[n|s|m|l|x]_[coco|obj365|obj2coco].yml`
- **Checkpoint format**: `.pth` PyTorch checkpoints; model weights downloadable from GitHub releases
