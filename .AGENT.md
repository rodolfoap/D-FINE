## Project overview

**D-FINE Inference** is a lightweight, inference-only implementation of D-FINE (Redefine Regression Task of DETRs as Fine-grained Distribution Refinement), based on the paper arXiv:2410.13842. This is a **fork stripped of all training code** — only the essentials for running object detection inference remain.

It's a real-time object detection framework supporting 5 model sizes (N/S/M/L/X) that redefine bounding box regression as Fine-grained Distribution Refinement (FDR). Models range from D-FINE-N (4M params, 42.8% AP) to D-FINE-X (62M params, 55.8% AP on COCO).

## Architecture

```
Input Image (JPG/PNG/BMP or MP4)
    ↓
Backbone (HGNetv2: N/S/M/L/X)
    ↓
Hybrid Encoder (fuses backbone features)
    ↓
D-FINE Decoder (transformer-based, FDR)
    ↓
Postprocessor (box decoding, NMS)
    ↓
Detection Output (boxes + classes + scores) → torch_results.jpg/mp4
```

**Key components:**
- **Backbone**: HGNetv2 family (B0-B5) — produces multi-scale feature maps
- **Encoder**: Hybrid encoder fuses backbone features via cross-scale attention
- **Decoder**: Transformer decoder with fine-grained distribution refinement
- **Postprocessor**: Decodes box predictions, filters by score, scales to original image size

## Usage commands

```bash
# Setup
conda create -n dfine python=3.11.9
conda activate dfine
pip install -r requirements.txt

# Quick inference (downloads L model, runs on horse.jpg)
./inference

# Custom image
python tools/inference/torch_inf.py \
  -c configs/dfine/dfine_hgnetv2_l_coco.yml \
  -r dfine_l_coco.pth \
  --input your_image.jpg \
  -d cuda:0

# Different model size
python tools/inference/torch_inf.py \
  -c configs/dfine/dfine_hgnetv2_n_coco.yml \
  -r dfine_n_coco.pth \
  --input your_image.jpg \
  -d cpu
```

## Conventions

- **Model selection**: Edit `inference` script or pass `-c` flag to choose model size
- **Device handling**: `-d cpu` (default) or `-d cuda:0` (GPU). Script prints which device is used.
- **Config composition**: YAML `__include__` merges base config (`dfine_hgnetv2.yml`) with model size overrides
- **Registry pattern**: `@register()` decorator auto-registers model classes during import
- **Checkpoint format**: `.pth` PyTorch checkpoints, auto-downloaded from GitHub releases on first run

## Critical areas

- **[src/zoo/dfine/dfine_decoder.py](src/zoo/dfine/dfine_decoder.py)**: Core FDR transformer decoder. Complex attention mechanics + distribution-based regression. Changes here affect detection accuracy.
- **[src/zoo/dfine/postprocessor.py](src/zoo/dfine/postprocessor.py)**: Box decoding and filtering. Handles coordinate transforms and confidence filtering.
- **[tools/inference/torch_inf.py](tools/inference/torch_inf.py)**: Main inference entry point. Device handling, image/video processing, input/output paths.
- **[configs/dfine/include/dfine_hgnetv2.yml](configs/dfine/include/dfine_hgnetv2.yml)**: Base model architecture config; all model variants include this. Defines layer counts, hidden dims, attention heads.
